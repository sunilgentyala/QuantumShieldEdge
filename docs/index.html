Abstract

What We Built and Why It Matters

The proliferation of IoT devices across smart homes, wearable healthcare systems, and connected vehicles has created an unprecedented demand for secure, privacy-preserving intelligence at the network edge. The classical cryptographic protocols that protect today's federated learning (FL) aggregation channels offer only computational security guarantees, which will not hold against a future quantum-capable adversary.

This paper presents QuantumShield-Edge, a hybrid quantum-classical architecture that embeds quantum key distribution (QKD) into the FL gradient aggregation pipeline for consumer IoT networks. The framework introduces a three-tier edge architecture in which QKD-generated one-time pad keys protect gradient transmissions, while a four-qubit variational quantum circuit (VQC) performs fingerprint-based anomaly detection on incoming model updates to flag Byzantine participants without ever exposing raw gradient data.

Simulation results on a 50-node heterogeneous IoT testbed demonstrate that QuantumShield-Edge achieves model accuracy within 1.3 pp of vanilla FedAvg while reducing gradient-poisoning attack success by 94.7% relative to state-of-the-art classical defenses. The framework tolerates quantum bit error rates of up to 11%, confirming its viability on emerging metropolitan QKD infrastructure.
System Design

Three-Tier Architecture

QKD transceivers co-located at each Edge Aggregation Node (EAN) and the cloud aggregator are connected via dark fiber or free-space optical links within a metropolitan QKD backbone. Post-quantum key encapsulation secures leaf-to-EAN traffic, seeded with QKD-derived material.

QKL

QKD Key Management Layer — Section IV-B

Runs decoy-state BB84 to bound photon-number-splitting attacks. Each EAN pre-loads 3 rounds of keying material. Proactive re-keying fires when buffer occupancy drops below 40%, fully decoupling QKD timing from FL round timing. Wegman-Carter MACs authenticate the classical post-processing channel without any computational hardness assumption.

QSAL

Quantum-Secured Aggregation Layer — Section IV-C

Top-K gradient sparsification at 5% density cuts key demand 20-fold, satisfying Equation (1) within current metropolitan QKD throughput. OTP encryption c = g ⊕ k delivers unconditional secrecy: no computational hardness assumption at any layer. Key material is consumed atomically and never reused.

VQAD

Variational Quantum Anomaly Detector — Section IV-D

Four gradient statistics are angle-encoded into a 4-qubit register via Ry rotations. Three alternating layers of parameterized Ry/Rz single-qubit rotations interleaved with CNOT brick-wall entanglement (48 trainable parameters total). Updates with ⟨Z⟩ < 0 are flagged Byzantine and down-weighted by 0.1 in the FedAvg aggregation step. Trained offline via the parameter-shift rule with Adam.

Evaluation

Simulation Results

Implemented in Python 3.11 with PennyLane 0.36, Flower (flwr) 1.8, and SimulaQron 3.0. 50-node Docker testbed on AMD Ryzen 9 7900X (64 GB RAM), 4 EAN clusters, CIFAR-10 with Dirichlet(α=0.5) non-IID partitioning. All metrics averaged over 5 seeds (std dev ≤ 0.3 pp accuracy, ≤ 0.8 pp detection rate).

Artifacts

Code & Experimental Artifacts

All source code, experiment configuration files, pre-trained VQAD weights, and raw result logs are publicly available in the GitHub repository. The manuscript is not included (IEEE copyright).

src/qkl/

Decoy-state BB84 emulator with QBER abort. Thread-safe key buffer with proactive re-keying at 40% occupancy.

src/qsal/

OTP encryptor (Equation 2). Top-K sparsifier at 5% density, cutting key demand 20-fold.

src/vqad/

4-qubit PennyLane VQC. 4-dim gradient fingerprint extractor. Parameter-shift + Adam offline trainer.

src/fl/

LightweightCNN (~180K params). Flower client with QKL. Flower server with VQAD soft-weighted FedAvg.

src/utils/attacks.py

Byzantine attack implementations: min-max gradient poisoning, label-flip, sign-flip.

experiments/

Simulation runner reproducing Table I. Offline VQAD training script. Full YAML configuration.

results/

Table I accuracy CSV. VQAD detection rate CSV. Key consumption and latency stats CSV.

models/vqad_weights.npy

Pre-trained VQAD circuit parameters (48 values). Load with detector.load_weights().
